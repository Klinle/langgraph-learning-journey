# 阶段 6: LangGraph 综合实战项目

完成了前五个阶段的基础与高级特性学习后，您已经具备了构建中大型 Multi-Agent 系统的硬实力。在这一阶段，您可以在本项目下独立编写各种实际业务场景 of Agent 解决方案。

## 推荐实战任务：多 Agent 协同客服系统

### 业务场景
模拟一个在线旅行平台的智能客服。系统由多个专注不同领域的子 Agent 构成，并共享一个全局状态。

### 系统架构
```text
                       [ 主路由 Agent (Router) ]
                              /      \
                             /        \
           [ 酒店预订 Agent ]          [ 机票预订 Agent ]
```

1. **主路由 Agent (Supervisor/Router)**:
   * 负责接收用户的输入，并通过 LLM 判定用户的意图。
   * 如果涉及酒店预订，将状态分发（或将执行流转）给“酒店预订 Agent”。
   * 如果涉及机票，流转给“机票预订 Agent”。
   * 任务完成后，重新接管对话。
2. **酒店预订 Agent (Sub-Agent 1)**:
   * 拥有专属的 Tools（查询房源、下单、退单）。
   * 负责完成闭环的酒店操作。
3. **机票预订 Agent (Sub-Agent 2)**:
   * 拥有专属的 Tools（查询航班、改签）。
   * 负责完成闭环的机票操作。

---

## 实践要点

* **状态双向同步**: 运用“子图”架构实现模块化，注意父子图状态字段的命名与数据传递。
* **人机协同审批**: 针对“下单/扣款”等敏感操作，必须配置 `interrupt_before` 中断，在主控台输出订单信息，待人工输入 `yes` 后再调用工具完成最终扣款。
* **链路追踪**: 开启 `.env` 中的 LangSmith 配置，运行后登录 LangSmith 平台查看复杂的路由流转图，极具视觉冲击力和调试便利度。
